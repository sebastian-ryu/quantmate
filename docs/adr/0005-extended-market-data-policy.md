# 0005. 확장 시장 데이터 저장 정책

상태: 결정

## 배경

현재 MVP는 전략 후보 검색과 백테스트를 일봉 중심으로 구현한다. 이후 단타, 분봉 전략, 실시간 감시, 공시 이벤트, 자동 주문 상태 동기화를 확장하려면 저장 기준을 먼저 정해두어야 한다.

## 결정

### 분봉 데이터

- 분봉은 `minute_prices` 계열 테이블로 분리한다.
- 기본 단위는 1분봉이며, 3분/5분/15분 등은 원본 1분봉에서 재계산한다.
- 저장 필드는 종목, 거래일시(한국시간), 시가, 고가, 저가, 종가, 거래량, 거래대금, 제공처, 수정 여부다.
- 초기 보관 기간은 최근 6개월로 제한한다. 장기 보관이 필요해지면 압축 또는 파티셔닝을 검토한다.
- 일봉 전략과 백테스트는 분봉 테이블을 직접 읽지 않는다. 분봉 전략이 추가될 때 별도 엔진에서 사용한다.

### 실시간 시세

- 실시간 체결과 호가는 영구 저장보다 장중 상태 관리가 우선이다.
- 실시간 원천 이벤트는 메모리 캐시 또는 짧은 TTL 저장소에 보관하고, 전략 신호나 주문 판단에 사용된 스냅샷만 DB에 남긴다.
- 저장 대상 스냅샷은 종목, 한국시간 시각, 현재가, 누적 거래량, 거래대금, 호가 요약, 제공처, 사용 목적이다.
- WebSocket 재연결, 중복 이벤트, 장 시작/장 종료 처리는 별도 수집 상태 테이블에 기록한다.
- 현재 구현은 KIS WebSocket 접속키를 발급/캐시하고, 국내주식 실시간체결가(`H0STCNT0`)를 서버 메모리 캐시에 수신하는 읽기 전용 연결이다. 실제 장중 수신 검증 후 호가, 체결통보, 주문 상태 동기화로 확장한다.

### 공시 데이터

- 공시는 `disclosure_events` 계열 테이블로 분리한다.
- 저장 필드는 종목, 공시 시각(한국시간), 제목, 공시 유형, 원문 URL, 제공처, 원천 ID, 요약, 중요도다.
- 원문 전문은 기본 저장하지 않고 URL과 요약을 저장한다. 전략에 필요한 키워드나 이벤트 분류만 구조화한다.
- OpenDART 또는 KRX 공시 연동 전까지 공시 기반 전략은 시스템 전략에 포함하지 않는다.

### 기업 이벤트

- 액면분할, 병합, 무상증자, 배당, 상장폐지, 거래정지는 `corporate_actions` 계열 테이블로 분리한다.
- 저장 필드는 종목, 이벤트 유형, 기준일, 효력 발생일, 비율/금액, 제공처, 원천 ID다.
- 백테스트는 가격 데이터의 수정주가 여부를 우선 사용하되, 기업 이벤트가 확보되면 수익률 계산 검증에 사용한다.
- 이벤트가 불완전한 기간은 백테스트 결과에 데이터 주의 문구를 표시한다.

### 시장 캘린더

- 미국 시장 확장 준비로 NYSE/Nasdaq 정규 휴장일 캘린더를 서버 유틸리티와 API로 제공한다.
- 현재 범위는 정규 휴장일과 주말 제외이며, 조기폐장과 한국 대체공휴일/음력 공휴일은 별도 데이터 공급원 확인 후 확장한다.

### KIS 체결통보와 주문 상태

- KIS WebSocket 체결통보는 주문 제출 결과의 최종 상태를 동기화하는 데 사용한다.
- 주문 요청은 이미 `broker_audit_logs`에 제출 전후로 남긴다.
- 체결통보가 들어오면 주문번호, 종목, 매수/매도, 주문수량, 체결수량, 체결가격, 거부 사유, 한국시간 수신 시각을 주문 상태 테이블에 반영한다.
- 주문 상태 동기화는 모의투자에서 먼저 검증하고, 실거래는 긴급 중지와 수동 확인 기능이 준비된 뒤 연결한다.

## 결과

- 일봉 기반 전략/백테스트와 장중/실시간 기능의 저장 책임이 분리된다.
- 대용량 분봉과 실시간 이벤트를 무작정 영구 저장하지 않는다.
- 공시와 기업 이벤트는 전략 신호에 쓰기 전 원천 ID와 제공처를 기준으로 추적한다.
- 한국시간 기준 저장 원칙을 유지한다.
