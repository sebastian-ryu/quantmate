# 0001. 초기 애플리케이션 구조

상태: 결정

## 배경

QuantMate는 한국 주식 종목 추천, 백테스트, 선택형 모의 투자를 우선 제공한다.

사용자는 Python을 다룰 수 있고, 전략/데이터/백테스트 생태계도 Python이 적합하다. 따라서 초기 MVP는 Java/Spring Boot 없이 Python 백엔드와 SvelteKit 웹 클라이언트로 시작한다.

## 결정

초기 구조는 다음과 같이 둔다.

- `backend/`: FastAPI 기반 로컬 API
- `frontend/`: SvelteKit 기반 웹 UI
- `docker-compose.yml`: 로컬 MySQL 8.4 LTS
- `docs/`: 프로젝트 규칙, 작업 목록, 결정 기록
- `AGENTS.md`: Codex 작업 지침

백엔드는 우선 샘플 전략, 추천 종목, 백테스트 미리보기 API를 제공한다. 실제 데이터 수집과 DB 저장은 다음 단계에서 붙인다.

프론트엔드는 리서치 대시보드부터 만든다. 첫 화면에서 전략 설명, 추천 후보, 백테스트 요약, 동작 모드를 확인할 수 있어야 한다.

웹 UI는 로컬 사용 도구이므로 초기에는 SvelteKit SSR을 끄고 브라우저에서 FastAPI를 호출하는 SPA 방식으로 둔다. 검색 노출보다 로컬 실행 안정성과 API 연동 단순성이 더 중요하다.

## 결과

- 초기 화면 개발을 백엔드와 함께 검증할 수 있다.
- 실제 증권 API 권한이 없어도 UI 흐름을 먼저 만들 수 있다.
- 실거래와 외부 API 연동은 안전장치와 권한 확인 이후 진행한다.
