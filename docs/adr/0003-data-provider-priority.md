# 0003. 데이터 공급원 우선순위

상태: 결정

## 배경

전략 검색, 백테스트, 향후 자동매매는 필요한 데이터 성격이 다르다.

- 전략 검색은 현재가, 수급, 재무비율, 공매도, 신용잔고, 순위 데이터가 필요하다.
- 백테스트는 장기간 일봉 가격과 종목 기본 데이터의 일관성이 중요하다.
- 자동매매는 실시간 시세, 계좌, 주문 API가 필요하다.

KRX 인증키 승인이 지연되고 있으므로 KRX만 기다리면 전략과 백테스트 개발이 멈춘다.

## 결정

초기 실제 데이터 연동은 한국투자증권 KIS Open API를 우선한다.

백테스트용 공식 장기 데이터는 KRX Open API 승인 후 KRX를 우선 사용한다.

KRX 승인이 지연될 때는 Yahoo/yfinance를 백테스트 개발용 임시 과거 가격 데이터로 사용한다. Yahoo/yfinance는 한국 주식 정식 공급원으로 고정하지 않는다.

데이터 접근 코드는 특정 API에 직접 의존하지 않고 다음 provider 인터페이스 뒤에 둔다.

- `MarketDataProvider`: 종목, 일봉, 분봉, 현재가
- `FundamentalProvider`: 재무비율, 실적, 배당
- `SupplyFlowProvider`: 외국인, 기관, 프로그램, 공매도, 신용잔고
- `BrokerProvider`: 계좌, 주문, 체결, 잔고

DB에는 제공처, 수정주가 여부, 수집 시점, 원천 기간을 저장한다. 전략과 백테스트는 API 응답이 아니라 정규화된 DB 데이터를 사용한다.

## 결과

- KRX 승인을 기다리지 않고 KIS로 전략 검색과 데이터 적재를 진행할 수 있다.
- KRX 승인이 완료되면 백테스트 공급원을 KRX로 바꿀 수 있다.
- Yahoo/yfinance로 임시 백테스트 엔진 개발을 진행하되, 결과에는 임시 데이터 사용 사실을 표시해야 한다.
- KIS와 KRX 데이터 차이를 추적할 수 있으므로 이후 데이터 품질 검증이 가능하다.
