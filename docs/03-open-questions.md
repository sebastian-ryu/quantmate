# 남은 질문

이 문서는 MVP 범위를 정하기 위한 질문과 이미 확정된 답변을 함께 기록한다.

## 확정된 답변

- 첫 MVP는 종목 추천과 백테스트를 우선 포함한다.
- 내부 가상 모의투자는 만들지 않는다. 모의투자는 이후 한국투자증권 모의투자 계좌 연동으로 진행한다.
- 제품은 장기적으로 단타, 스윙, 중기, 장기 퀀트 스타일을 모두 지원한다.
- 첫 구현은 웹 전략 빌더보다 여러 기본 전략 제공을 먼저 한다.
- 구현 우선순위는 전략, 백테스트, 검색기 순서다.
- 자동 주문과 실거래는 종목 선정 정확도와 데이터 최신성 유지가 충분히 검증된 뒤 마지막 단계로 미룬다.
- 웹 UI 조건 빌더는 자유도가 낮게 시작하더라도 나중에 제공하는 방향이 좋다.
- 저장소는 https://github.com/sebastian-ryu/quantmate 에 있다.
- 사용자에게 전달하는 요약은 간결하게 한다.
- 가능하면 질문은 한 번에 하나씩 한다.
- 전략/데이터/백테스트가 Python을 쓰므로 백엔드도 Python이 우선 후보가 되었다.
- MySQL은 Docker Compose로 실행한다.
- 증권 관련 API에서 접근 권한, 계좌 신청, 토큰, 인증 정보가 필요하면 사용자와 문답하며 권한을 확보한 뒤 진행한다.

## 매매 범위

1. 확정: 첫 사용 가능 버전은 종목 추천과 백테스트를 포함한다.
2. 확정: 내부 시뮬레이션 모의투자는 만들지 않고, 이후 한국투자증권 모의투자/실거래 API를 연동한다.
3. 실거래를 하게 된다면 현금 주식 주문만 지원할지, ETF/신용/파생까지 볼지 결정 필요.

## 투자 스타일

4. 확정: 장기적으로 단타, 1-10일 스윙, 수주-수개월, 장기 팩터 투자를 모두 지원한다.
5. 확정: 첫 알고리즘은 퀀트/팩터와 트레이더형 전략을 함께 제공하는 혼합형으로 시작한다.
6. 제안: 초기 종목 추천은 일봉/장마감 데이터 기준으로 만들고, 장중/분봉 전략은 실시간 데이터 연동 뒤 확장한다.

## 데이터

7. 확정: 초기 실제 데이터는 KIS Open API와 KRX Open API를 함께 사용한다. 백테스트용 공식 장기 일봉은 KRX를 우선 사용하고, KRX 커버리지가 부족한 구간은 KIS/Yahoo로 보완한다.
7-1. 완료: KRX 종목기본정보와 일별매매정보 서비스 접근 권한을 확보했고, 실제 호출과 DB 저장을 검증했다.
8. 분봉 데이터가 초기부터 필요한지 결정 필요.
9. 틱/호가 데이터를 저장할지, 신호 계산에만 임시 사용 할지 결정 필요.
10. PER/PBR/ROE, 재무제표, 배당, 이익 성장, 애널리스트 추정치, 공시 중 어떤 재무 데이터가 먼저 필요한지 결정 필요.

## 전략 작성

11. MVP 확정: 우선 여러 기본 전략을 제공한다.
12. 이후 확정: 기본 전략과 백테스트가 안정화된 뒤 제한적인 웹 UI 조건 빌더를 추가한다.
13. 초기 시스템 제공 전략 묶음은 사용자 확인 후 확정한다.

## 로컬 환경

13. 확정: MySQL과 로컬 서비스는 Docker Compose를 사용한다.
14. 나중에 `./dev` 또는 `make dev` 같은 단일 실행 명령을 만들지 결정 필요.
15. 주 개발 및 실행 OS를 기록할 필요가 있다.

## 증권사와 계좌

16. 한국투자증권 KIS API 신청 가능한 계좌가 있는지 확인 필요.
17. 확정: 첫 증권사 연동은 KIS 읽기 전용 데이터 조회부터 시작한다. 주문 기능은 모의투자 계좌가 준비되어 있어도 현재 우선순위에서 뒤로 미루고, 종목 선정 정확도와 최신성 유지가 충분히 검증된 뒤 진행한다.
18. 실거래 기능 전에 기본 주문 제한값을 어떻게 둘지 결정 필요.
19. API 권한 신청이나 인증 정보 발급이 필요한 시점에는 사용자에게 필요한 작업을 한 가지씩 질문하며 진행한다.

## GitHub

20. 확정: 저장소는 public 공개하지 않고 개인 비공개 저장소로 유지한다.
21. GitHub 인증 방식은 현재 로컬 push가 가능한 방식으로 유지한다.

## UI 방향

22. 확정: 첫 화면은 종목 검색기다.
23. 대시보드는 실시간 데이터, 계좌 상태, 운영 요약이 필요해지는 시점에 다시 검토한다.
