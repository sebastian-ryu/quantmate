# 초기 결정 사항

각 항목은 검토 전까지 `제안` 상태로 둔다.

## 기술 스택

### 백엔드

상태: 결정

메인 백엔드는 Python + FastAPI를 사용한다.

이유:

- 이 프로젝트는 데이터 수집, 전략 계산, 백테스트에 Python이 필요하다.
- 백엔드까지 Python으로 맞추면 Java/Python 연동 복잡도를 초기에 줄일 수 있다.
- FastAPI는 로컬 API 서버, 비동기 증권사 호출, 스케줄 작업, 타입 기반 API 계약에 적합하다.
- 증권사 API 키와 매매 안전장치는 브라우저가 아니라 서버 쪽에 있어야 한다.

Java + Spring Boot는 서버가 더 큰 엔터프라이즈형 애플리케이션으로 성장할 때 대안으로 다시 검토한다.

### 퀀트 실행 영역

상태: 결정

처음에는 데이터 분석, 전략 계산, 백테스트 모듈을 백엔드 코드베이스 안의 Python 코드로 구현한다.

이유:

- Python은 주식 데이터, 수치 계산, 백테스트 생태계가 강하다.
- 사용자 정의 전략을 Python으로 먼저 실험하기 쉽다.
- 전략 로직을 웹 UI와 분리해 테스트하기 쉽다.

나중에 데이터 수집이나 백테스트가 API 프로세스에 부담을 주면 별도 워커로 분리한다.

### 웹 클라이언트

상태: 결정

SvelteKit + TypeScript를 사용한다.

이유:

- 로컬 검색기, 전략 설정 폼, 결과 화면을 만들기 좋다.
- 구현을 작은 단계로 나누면 사용자가 SvelteKit을 깊이 몰라도 진행할 수 있다.
- UI는 데이터를 보여주고, 전략을 설정하고, 작업을 실행하고, 결과를 확인하는 얇은 계층으로 유지할 수 있다.

### 데이터베이스

상태: 결정

MySQL 8.4 LTS를 Docker Compose로 로컬에서 실행한다.

이유:

- MySQL은 사용자가 고려하던 선택지다.
- MySQL 8.4는 LTS 계열이라 장기 로컬 프로젝트에 적합하다.
- Docker Compose를 쓰면 설치 환경을 반복 가능하게 만들고, PC별 설정 차이를 줄일 수 있다.

직접 설치 방식은 기본 경로로 사용하지 않는다.

나중에 검토할 수 있는 추가 선택지:

- 빠른 연구용 스냅샷을 위한 DuckDB 또는 Parquet
- MySQL이 시계열 처리 병목이 될 경우 PostgreSQL/TimescaleDB

### 데이터와 증권사 API

상태: 결정

초기 실제 데이터 연동은 한국투자증권 KIS Open API를 우선 사용한다. KIS는 현재가, 일/주/월/년 기간별 시세, 분봉, 재무비율, 수급/공매도/신용잔고/프로그램 매매, 순위분석, 실시간 시세, 계좌/주문까지 한 흐름으로 이어지므로 전략 검색과 향후 자동매매를 가장 빨리 연결할 수 있다.

초기 사용 순서:

1. OAuth/토큰과 설정 처리
2. 국내 주식 종목 정보와 현재가/기간별 시세 조회
3. 전략에 필요한 재무비율, 수급, 공매도, 신용잔고, 순위 데이터 조회
4. WebSocket 실시간 시세 수집
5. 계좌 읽기 전용 API
6. 한국투자증권 모의투자 계좌 주문 API
7. 한국투자증권 실거래 주문 API

### 과거 데이터 소스

상태: 결정

데이터 종류에 따라 여러 소스를 사용한다.

- KIS Open API: 초기 실제 데이터, 전략 검색, 수급/재무/실시간, 향후 계좌/매매
- KRX Open API: 승인 후 백테스트용 공식 장기 일봉과 종목 기본 데이터
- Yahoo/yfinance: KRX 승인 지연 시 백테스트 개발용 임시 과거 가격 데이터, 미국 시장 확장 후보
- OpenDART: 재무제표, 공시, 기본적 분석 데이터
- pykrx/FinanceDataReader: 검증 또는 임시 보조 후보

가져온 데이터에는 제공처 정보를 함께 저장해 나중에 KIS, KRX, Yahoo 데이터 차이를 추적할 수 있게 한다. 백테스트 엔진은 특정 API가 아니라 정규화된 DB 테이블을 보게 만들어 KRX 승인 후 빠르게 공급원을 바꿀 수 있게 한다.

## 초기 전략 묶음

상태: 제안

기본 제공 전략은 설명 가능하고 백테스트 가능한 전략부터 시작한다.

초기 후보 전략:

- 상대 모멘텀: 최근 3~12개월 상대 강도, 이동평균 추세, 거래대금 필터
- 가치/퀄리티 팩터: PER, PBR, ROE, 부채비율, 이익 안정성, Piotroski형 재무 점수
- 성장 돌파형: CAN SLIM을 참고한 이익 성장, 신고가 근접, 거래량 증가, 시장 주도주 조건
- 추세 돌파형: Donchian/Turtle류 가격 채널 돌파, 변동성 기반 손절/청산 규칙
- 수급 누적형: 외국인/기관 순매수, 거래대금 증가, 가격 추세 확인
- 저변동성/방어형: 유동성 있는 종목 중 변동성, 낙폭, 재무 안정성을 함께 보는 전략

초기 구현은 일봉 OHLCV와 기본 재무/수급 데이터로 가능한 전략부터 만든다. 분봉, VWAP, 장중 급등 포착 전략은 데이터 수집과 실시간 연동이 준비된 뒤 추가한다.

어떤 전략도 항상 좋은 전략으로 취급하지 않는다. 각 전략은 다음 정보를 가져야 한다.

- 의도
- 필요한 데이터
- 신호 규칙
- 위험과 실패 사례
- 기본 파라미터
- 백테스트 가정
