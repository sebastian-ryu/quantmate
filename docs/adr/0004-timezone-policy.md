# 0004. 시간대 정책

상태: 결정

## 배경

QuantMate의 첫 대상 시장은 한국 주식이다. 사용자 입력, 화면 표시, 거래일, DB 저장 시간이 서로 다른 시간대를 사용하면 백테스트 기간, 수집 일자, 거래일 해석이 어긋날 수 있다.

## 결정

앱의 기본 시간대는 `Asia/Seoul`이다.

- 사용자가 입력하는 날짜와 시간은 한국시간으로 해석한다.
- 화면에 표시하는 날짜와 시간은 한국시간으로 표시한다.
- DB에 저장하는 업무 시간값은 한국시간 기준으로 저장한다.
- 거래일은 한국 시장 기준 거래일로 표현한다.
- UTC는 외부 API 호출, 외부 타임스탬프 변환, 내부 기술적 변환이 필요한 경우에만 사용한다.
- UTC를 쓰더라도 사용자에게 노출되는 값은 한국시간으로 변환한다.

## 결과

- 사용자는 항상 한국시간을 쓰는 것처럼 느낀다.
- Yahoo 같은 외부 API 타임스탬프도 한국 거래일로 변환한 뒤 저장/표시한다.
- MySQL 컨테이너는 `TZ=Asia/Seoul`, `--default-time-zone=+09:00`으로 실행한다.
- Python 코드에서는 공통 시간 유틸을 통해 한국시간을 생성하고 변환한다.
