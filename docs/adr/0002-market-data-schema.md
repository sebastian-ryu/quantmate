# 0002. 초기 시장 데이터 스키마

상태: 결정

## 배경

종목 추천과 백테스트를 하려면 최소한 시장, 종목, 일봉 가격, 데이터 수집 이력이 필요하다.

초기에는 한국 주식과 일봉 데이터를 먼저 다룬다. 분봉, 실시간, 공시 데이터는 이후 확장한다.

## 결정

초기 DB 스키마는 다음 테이블로 시작한다.

- `markets`: 시장 정보
- `instruments`: 종목 정보
- `daily_prices`: 일봉 OHLCV와 제공처 정보
- `quote_snapshots`: 현재가 조회 시점의 가격, 거래량, 거래대금, PER/PBR, 시가총액, 회전율 등 보조 필드
- `fundamental_ratios`: KIS 재무비율의 매출/영업이익/순이익 성장률, ROE, EPS, BPS, 부채비율
- `supply_flow_dailies`: KIS 투자자별 매매동향의 외국인, 기관, 연기금 일별 순매수 수량과 금액
- `risk_indicator_dailies`: KIS 공매도와 신용잔고 일별 리스크 보조 지표
- `data_import_jobs`: 데이터 수집 작업 이력

마이그레이션은 Alembic으로 관리한다. ORM은 SQLAlchemy 2 계열을 사용한다.

## 결과

- 데이터 제공처와 수정주가 여부를 가격 데이터에 함께 저장할 수 있다.
- 동일 종목, 거래일, 제공처, 수정주가 여부 조합은 중복 저장하지 않는다.
- 동일 종목, 조회일, 제공처 조합의 현재가 스냅샷은 중복 저장하지 않는다.
- 동일 종목, 결산기간, 기간 유형, 제공처 조합의 재무비율은 중복 저장하지 않는다.
- 동일 종목, 거래일, 제공처 조합의 일별 수급 데이터는 중복 저장하지 않는다.
- 동일 종목, 거래일, 제공처 조합의 공매도/신용잔고 리스크 데이터는 중복 저장하지 않는다.
- 실제 MySQL 8.4 컨테이너에 초기 마이그레이션을 적용해 검증했다.
