# 0003. 데이터 공급원 우선순위

상태: 결정

## 배경

전략 검색, 백테스트, 향후 자동매매는 필요한 데이터 성격이 다르다.

- 전략 검색은 현재가, 수급, 재무비율, 공매도, 신용잔고, 순위 데이터가 필요하다.
- 백테스트는 장기간 일봉 가격과 종목 기본 데이터의 일관성이 중요하다.
- 자동매매는 실시간 시세, 계좌, 주문 API가 필요하다.

KRX 인증키 승인이 지연되고 있으므로 KRX만 기다리면 전략과 백테스트 개발이 멈춘다.

## 결정

초기 실제 데이터 연동은 한국투자증권 KIS Open API를 우선한다.

백테스트용 공식 장기 데이터는 KRX Open API 승인 후 KRX를 우선 사용한다.

KRX 승인이 지연될 때는 KIS 일봉을 먼저 수집하고, KIS가 충분히 커버하지 못하는 종목과 기간은 Yahoo/yfinance로 보완한다. Yahoo/yfinance는 한국 주식 정식 공급원으로 고정하지 않는다.

데이터 접근 코드는 특정 API에 직접 의존하지 않고 다음 provider 인터페이스 뒤에 둔다.

- `MarketDataProvider`: 종목, 일봉, 분봉, 현재가
- `FundamentalProvider`: 재무비율, 실적, 배당
- `SupplyFlowProvider`: 외국인, 기관, 프로그램, 공매도, 신용잔고
- `BrokerProvider`: 계좌, 주문, 체결, 잔고

DB에는 제공처, 수정주가 여부, 수집 시점, 원천 기간을 저장한다. 전략과 백테스트는 API 응답이 아니라 정규화된 DB 데이터를 사용하며, 같은 실행 안에서도 종목별로 `KRX -> KIS -> Yahoo` 우선순위에 따라 공급원을 혼합할 수 있다. 단, 백테스트는 공급원 이름보다 요청 기간 커버리지를 우선한다. 예를 들어 KIS 일봉이 최근 일부 기간만 있고 Yahoo 일봉이 요청 기간을 더 길게 덮으면 백테스트는 Yahoo를 선택한다.

KIS 현재가 응답에서 바로 확보 가능한 PER/PBR/시가총액/회전율 등은 `quote_snapshots`에 저장한다. 가격 자체는 일봉 종가를 우선 사용하고, 현재가 스냅샷은 검색기와 전략 후보의 보조 필드 보강에만 사용한다.

KIS 재무비율은 `fundamental_ratios`에 저장하고, 최신 연간 결산 기준의 ROE, 매출/영업이익/순이익 성장률, 부채비율을 검색기와 전략 후보의 재무 필드에 반영한다.

KIS 시가총액 상위 랭킹은 전략 후보 유니버스를 넓히는 시드로 사용한다. KIS 토큰 제한이나 일시 오류로 랭킹 호출이 실패하면, 이미 `daily_prices`에 저장된 종목을 거래대금 기준 보조 시드로 사용해 검색기 후보 풀이 기본 12개로 축소되지 않게 한다.

KIS 투자자별 매매동향 일별 데이터는 `supply_flow_dailies`에 저장하고, 외국인/기관/연기금 5일 및 20일 순매수 금액을 검색기와 전략 후보의 수급 필드에 반영한다.

KIS 공매도 일별추이와 신용잔고 일별추이는 `risk_indicator_dailies`에 저장하고, 공매도 비중과 신용잔고 단기 변화율을 검색기와 전략 후보의 리스크 필드에 반영한다.

## 결과

- KRX 승인을 기다리지 않고 KIS로 전략 검색과 데이터 적재를 진행할 수 있다.
- KRX 승인이 완료되면 백테스트 공급원을 KRX로 바꿀 수 있다.
- KIS와 Yahoo/yfinance를 혼합해 임시 백테스트 엔진 개발을 진행하되, 결과에는 사용된 데이터 공급원을 표시해야 한다.
- 검색기 API는 전략 후보 일봉 데이터와 KIS 현재가/재무비율/수급/공매도/신용잔고 데이터를 조합해 먼저 동작시키고, 부족한 재무 상세 데이터는 KRX/OpenDART 매핑 후 보강한다.
- 검색기는 전략 화면보다 넓은 후보 풀을 요청하며, KIS 랭킹과 저장된 일봉 종목을 함께 사용해 실제 데이터 후보를 확장한다.
- KIS와 KRX 데이터 차이를 추적할 수 있으므로 이후 데이터 품질 검증이 가능하다.
