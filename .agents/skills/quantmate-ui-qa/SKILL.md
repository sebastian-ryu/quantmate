# QuantMate UI 검토

## 원칙

- 메뉴는 `검색기`, `전략`, `백테스트` 중심으로 유지한다.
- 검색기는 상단 조건식, 하단 결과 구조를 유지한다.
- 전략은 전략 선택 영역을 먼저 보여주고, 후보/주문/실시간 데이터는 로딩되는 대로 표시한다.
- 백테스트는 전략 선택, 조건, 실행 버튼, 결과 블록 순서로 한 줄에 하나의 블록을 배치한다.
- 버튼과 폰트 밀도는 검색기 페이지와 맞춘다.
- 기능 설명 문구를 화면에 길게 늘어놓지 말고, 필요한 설명은 툴팁에 둔다.

## 확인 순서

1. `frontend/src/lib/api.ts` 타입이 백엔드 응답과 맞는지 본다.
2. 해당 Svelte 페이지와 `frontend/src/app.css`를 함께 수정한다.
3. `npm --prefix frontend run check`를 실행한다.
4. 로컬 서버가 떠 있으면 인앱 브라우저에서 직접 화면을 확인한다.
5. 인앱 브라우저가 동작하지 않을 때만 Playwright나 Chrome 도구를 사용한다.

## 화면별 체크

- 검색기: 필터 선택 수, 활성 필터 칩, 미연동 조건, 결과 테이블 스크롤을 확인한다.
- 전략: 시스템/사용자 전략 구분, 후보 10개 이상, 컬럼 툴팁, 일괄 주문 제안, KIS 실시간 시세 상태를 확인한다.
- 백테스트: 실행 로딩, 자산 성장 차트 hover 툴팁, 비교군 성과 요약, 최근 백테스트 재조회 차트가 같은 모양인지 확인한다.
