# 0002. 초기 시장 데이터 스키마

상태: 결정

## 배경

종목 추천과 백테스트를 하려면 최소한 시장, 종목, 일봉 가격, 데이터 수집 이력이 필요하다.

초기에는 한국 주식과 일봉 데이터를 먼저 다룬다. 분봉, 실시간, 재무, 수급 데이터는 이후 확장한다.

## 결정

초기 DB 스키마는 다음 테이블로 시작한다.

- `markets`: 시장 정보
- `instruments`: 종목 정보
- `daily_prices`: 일봉 OHLCV와 제공처 정보
- `data_import_jobs`: 데이터 수집 작업 이력

마이그레이션은 Alembic으로 관리한다. ORM은 SQLAlchemy 2 계열을 사용한다.

## 결과

- 데이터 제공처와 수정주가 여부를 가격 데이터에 함께 저장할 수 있다.
- 동일 종목, 거래일, 제공처, 수정주가 여부 조합은 중복 저장하지 않는다.
- 실제 MySQL 8.4 컨테이너에 초기 마이그레이션을 적용해 검증했다.
