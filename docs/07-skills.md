# 스킬 운영 계획

## 목적

이 문서는 QuantMate 프로젝트에서 어떤 작업을 Codex 스킬로 만들지 관리한다.

처음부터 스킬을 많이 만들지 않는다. 반복 작업이 생기거나 절차가 복잡해졌을 때 실제 스킬로 승격한다.

## 스킬로 만들 기준

아래 중 하나에 해당하면 스킬 생성을 검토한다.

- 같은 절차를 여러 번 반복한다.
- API 인증, 데이터 수집, 백테스트처럼 실수 비용이 큰 절차다.
- 실행 순서가 중요하고 중간 검증이 필요하다.
- 프로젝트 고유 규칙을 매번 길게 설명해야 한다.
- 참고 문서나 스크립트를 함께 묶어두면 작업 품질이 올라간다.

## 초기 후보 스킬

### 증권사 API 연동 스킬

상태: 후보

사용 시점:

- 한국투자증권 Open API 인증 흐름을 구현할 때
- 토큰 발급/갱신, WebSocket 접속키, 계좌 읽기 권한을 다룰 때
- API 권한 신청이나 인증 정보 발급이 필요할 때

핵심 규칙:

- 사용자의 확인 없이 실계좌 주문 권한을 사용하지 않는다.
- 인증 정보는 `.env`에만 둔다.
- 필요한 권한은 사용자와 한 가지씩 문답하며 확보한다.

### 데이터 수집 스킬

상태: 후보

사용 시점:

- KRX, FinanceDataReader, pykrx, OpenDART, KIS 데이터를 수집할 때
- 데이터 제공처별 차이를 검증할 때
- 수집 이력과 제공처 메타데이터를 저장할 때

핵심 규칙:

- 데이터 출처와 수집 시간을 저장한다.
- 수정주가 여부를 명확히 기록한다.
- 호출 제한을 고려한다.

### 전략과 백테스트 스킬

상태: 후보

사용 시점:

- 기본 전략을 추가할 때
- 전략 설명, 필요 데이터, 실패 조건을 문서화할 때
- 백테스트 지표와 가정을 표준화할 때

핵심 규칙:

- 전략은 항상 의도, 필요 데이터, 신호 규칙, 위험을 함께 가진다.
- 백테스트는 거래 비용, 슬리피지, 리밸런싱 가정을 기록한다.

### UI 검토 스킬

상태: 후보

사용 시점:

- 화면 구성을 여러 안으로 비교할 때
- 참고 제품을 검토하고 QuantMate에 맞는 화면 방향을 정할 때

핵심 규칙:

- 첫 화면은 실제 사용 화면이어야 한다.
- 마케팅형 랜딩 페이지보다 리서치와 실행 흐름을 우선한다.

### 문서 최신화 스킬

상태: 후보

사용 시점:

- 기능 구현 후 관련 문서를 갱신할 때
- 작업 목록 상태를 업데이트할 때
- 설계 변경 영향 범위를 확인할 때

핵심 규칙:

- 구현 변경과 문서 변경을 분리하지 않는다.
- 설계가 바뀌면 작업 목록과 결정 문서를 함께 확인한다.

## 실제 스킬 생성 위치

저장소에 함께 관리할 실제 Codex 스킬이 필요해지면 아래 형태를 사용한다.

```text
.agents/skills/
  skill-name/
    SKILL.md
    references/
    scripts/
```

스킬은 작고 명확하게 만든다. 단순 문서나 일회성 절차는 스킬로 만들지 않는다.

## 스킬 외 Codex 설정 후보

아래 항목은 당장 만들지 않고, 필요해지는 시점에 추가한다.

### `.codex/config.toml`

상태: 보류

용도:

- 프로젝트 단위 Codex 설정
- 프로젝트 문서 인식 범위 조정
- 프로젝트 로컬 훅 연결

보류 이유:

- 현재는 `AGENTS.md`와 문서 규칙만으로 충분하다.
- 설정 파일은 프로젝트 신뢰 설정과 연결되므로 필요할 때 작게 추가한다.

### `.codex/hooks.json`

상태: 보류

용도:

- 커밋 전 검사 강제
- 비밀 정보 커밋 방지 검사
- 작업 종료 시 문서 업데이트 누락 확인

보류 이유:

- 아직 애플리케이션 코드와 테스트 명령이 없다.
- 훅은 신뢰 검토가 필요한 실행 코드이므로 실제 반복 문제가 생긴 뒤 추가한다.

### 사용자 전역 Codex 설정

상태: 사용자 선택

용도:

- 모든 저장소에 적용할 개인 기본 규칙
- 응답 길이, 승인 정책, 모델 선호 등 개인 설정

보류 이유:

- 이 저장소에 커밋할 내용이 아니다.
- 필요하면 사용자 PC의 `~/.codex` 아래에서 별도로 관리한다.
